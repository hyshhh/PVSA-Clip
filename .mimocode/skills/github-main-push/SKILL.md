# GitHub 主干直推技能

在这个仓库里执行 GitHub 推送相关操作时，遵循下面规则。

## 默认策略

1. 把 `main` 视为默认目标分支。
2. 用户说“推送”时，默认含义是把当前有效改动直接落到远端 `main`。
3. 不要默认创建功能分支。
4. 不要默认保留临时分支。
5. 不要默认发起合并请求。

## 执行流程

1. 先检查工作区是否干净，并确认哪些改动需要提交。
2. 如果改动已经在临时分支上，先把改动合并到本地 `main`，再推送 `main`。
3. 推送前先同步远端 `main`，避免把过期本地主干直接推上去。
4. 推送完成后，若存在这次操作临时创建的功能分支，删除本地和远端分支。

## 风险控制

1. 只有用户明确要求时，才推送到非 `main` 分支。
2. 只有用户明确说“强推”“覆盖历史”“改写远端历史”时，才使用强制推送。
3. 如果远端 `main` 有新提交，优先快进同步；不能快进时，完成普通合并后再推送，除非用户明确要求改写历史。

## 输出要求

1. 明确说明最终落到哪个远端、哪个分支、对应提交号。
2. 如果发生了合并，要说明是先同步主干再合并推送。
3. 如果删除了临时分支，要同时说明本地和远端删除结果。
