# Git 推送命令

推送代码到 GitHub 仓库。这个仓库里，用户说“推送”时，默认含义是把当前完成的改动直接提交并推送到远端 `main`，而不是创建功能分支。

## 使用方法

```
/push $ARGUMENTS
```

`$ARGUMENTS` 格式：`[远程仓库名] [分支名] [--force]`。如果不指定参数，默认推送到 `pvsa-clip` 的 `main`。

## 实现

1. 检查当前 Git 状态。
2. 如果当前不在 `main`，先把当前已完成改动合并到本地 `main`，不要保留临时功能分支作为默认结果。
3. 推送代码到指定的远程仓库和分支；未显式指定时，直接推送到 `pvsa-clip/main`。
4. 自动添加 `-c http.sslVerify=false` 以跳过 SSL 验证（解决 Windows Schannel TLS 握手问题）。
5. 只有用户明确指定 `--force`，或明确说“强推”“覆盖远端历史”时，才添加 `--force`。
6. 报告推送结果。

## 注意事项

- 本机 Windows Schannel TLS 握手有问题，必须使用 `-c http.sslVerify=false` 跳过 SSL 验证。
- SSH 端口 22 不通，不要尝试 SSH 方式。
- 用户说“推送”时直接推，不要问确认。
- 除非用户明确要求，不要新建功能分支，不要保留功能分支，不要把“推送”默认理解成发起合并请求。
- 用户说“推送就是覆盖 main”时，默认理解为把最新有效改动落到远端 `main`；只有用户明确要求改写历史时，才允许强制推送。
